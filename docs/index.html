Why pcloud-cli

Features

Everything you need to manage your pCloud storage from the terminal — no browser required after the one-time OAuth login.

🔐
OAuth2 Login
One-time browser-based authentication via pCloud's OAuth2 authorization code flow. Your password never touches the CLI.
🗑️
Trash Listing
View all files in your pCloud trash with full metadata — name, path, deletion date, and file size.
♻️
Restore from Trash
Recover any deleted file instantly by its ID. No hunting through the web UI required.
⏪
Rewind History
Browse the full version history of any file path using pCloud's built-in rewind feature.
🔄
Restore from Rewind
Roll back any file to a previous version and save it to a destination path of your choosing.
💾
Token Storage
Access tokens are persisted locally so you stay authenticated across sessions without re-logging in.
Get going fast

Quick Start

Install, authenticate once, and start managing your files in under a minute. 

# Install globally
npm install -g @kud/pcloud-cli

# Add your OAuth credentials to .env
echo "PCLOUD_CLIENT_ID=your_client_id" >> .env
echo "PCLOUD_CLIENT_SECRET=your_client_secret" >> .env

# Authenticate once — opens browser for OAuth
pcloud login
🔐 Opening browser for authentication...
✓ Authentication successful!
🎉 Setup complete! Your access has been saved securely.

# List files in trash
pcloud list-trash

Files in trash:
================

File ID: 12345
Name: report.pdf
Path: /Documents/report.pdf
Deleted: 4/17/2026, 10:32:00 AM
Size: 1.24 MB

# Restore a file
pcloud restore-trash 12345
✓ Successfully restored file 12345
Commands

CLI Reference

All commands available via the pcloud binary.

pcloud login Authenticate with pCloud via OAuth2. Opens your browser once — saves the token locally for all future commands.
pcloud logout Remove the stored access token from your machine.
pcloud list-trash List all files in your pCloud trash with name, path, deletion date, and size.
pcloud restore-trash <fileid> Restore a deleted file from trash by its numeric file ID.
pcloud list-rewind <path> List all rewind events (version history) for a file path, e.g. /mydocument.txt.
pcloud restore-rewind <fileid> <topath> Restore a previous version of a file by its ID to a destination path.
Configuration

Configuration

Set environment variables in a .env file at the project root, or export them directly in your shell.

Key Required Default Description
PCLOUD_CLIENT_ID Yes — Your pCloud OAuth app client ID from the pCloud developer console.
PCLOUD_CLIENT_SECRET Yes — Your pCloud OAuth app client secret, used to exchange the authorization code for an access token.
PCLOUD_ACCESS_TOKEN No — Optional direct access token. If set, bypasses the stored token and OAuth flow entirely.
